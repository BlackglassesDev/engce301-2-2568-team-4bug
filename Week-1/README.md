## DevTools Reflection (Week 1)
1) Browser รู้ได้อย่างไรว่าต้องรัน JavaScript ใน `<script>` เมื่อโหลดหน้าเว็บ?
   → โดยปกติแล้วเมื่อเราทำการเปิดไฟล์เพื่อรันบน Browser เพื่อแสดงผลหน้าเว็บนั้น Browser จะมีการทำงานโดยเริ่มอ่านหรือรันโค้ดที่เราได้เขียนลงไปจากซ้ายไปขวาและบนลงล่างทีละบรรทัด และเมื่อเจอแท็ก `<script>` มันก็จะหยุดการรันอย่างอื่นและส่งโค้ดไปให้ javascript ประมวลผลต่อ เมื่อทำเสร็จจึงกลับมารันในส่วนของ Code อื่นๆต่อ

2) ถ้าไม่มี DevTools การแก้ไขปัญหา (debug) เวลาเว็บมีปัญหาจะยากขึ้นอย่างไร?

-  หาสาเหตุของ Error ยากขึ้น ไม่สามารถเจาะจงเฉพาะปัญหาได้ เช่น เกิดปัญหาที่ Javascript
- ไม่สามารถเห็นโครงสร้าง Html/Css ของหน้าเว็บจริงได้ ทำให้เมื่อเกิดปัญหา U เพี้ยนก็จะหาจุดยากขึ้น
- ไม่สามารลองแก้ไข Css บนหน้าเว็บจริงแบบ Realtime ได้ ต้องเสียเวลากับแก้โค้ดและรันไปมาซ้ำๆ

3) แต่ละคนคิดว่า Panel ไหนใน DevTools จะได้ใช้บ่อยที่สุดในเทอมนี้ (Elements / Console / Network / อื่น ๆ)? เพราะอะไร? 

- (ชโนดม อองกุลนะ) ชอบ Console เพราะใช้ดู log ของ javaScript และทำให้รู้แนวทางการทำงานรวมถึงเช็คการทำงานของโปรแกรมได้ 
- (ชนาธิป ระวิมี) Elements ใช้จัดการ layout ต่างๆ และ จัดการ Debug CSS
- (จิดาภา กันทวงศ์) น่าจะ Elements คิดว่าจะได้ใชบ่อยสุดในเทอมนี้เพราะชอบใช้ส่วนนี้ในการแก้หน้าเว็บและศึกษาว่าแก้แล้วจะออกมาเป็นอย่างไร
- (อรนุช ลุงหลิ่ง) ชอบ Elements เพราะมักจะมีปัญหาในส่วนของ UI CSS บ่อย และชอบใช้ส่วนนี้ในการแก้ปัญหาแต่ละครั้งไม่ว่าจะเป็นปัญหาในเรื่อง เช่น การจัดวางที่เพี้ยน หรือเจอบัคโค้ดไม่แสดงบนเว็บจริง
